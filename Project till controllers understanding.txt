# 1. Why SQL table shows fewer columns but model has more fields?

## Question

Why does SQL Server table show only 5 columns, but model has extra properties like `Office` and `Trips`?

## What it is

SQL table contains only actual database columns:

* `bus_id`
* `office_id`
* `registration_number`
* `capacity`
* `type`

But EF Core also generated:

```csharp id="b8j1y2"
public virtual AgencyOffice Office { get; set; } = null!;
public virtual ICollection<Trip> Trips { get; set; } = new List<Trip>();
```

These are called **Navigation Properties**.

---

## Why we use it / Purpose

They help EF Core understand relationships between tables and allow easy navigation between related data.

Example:

```csharp id="f1v4n8"
bus.Office
```

gets related office data.

```csharp id="w2m8q1"
bus.Trips
```

gets all trips of that bus.

These properties are not physical DB columns.

# 3. Why `public virtual`?

## Question

Why EF Core generated `virtual` keyword?

## What it does

`virtual` allows EF Core to create proxy classes for:

* Lazy Loading
* Runtime overriding
* Better testing/mocking

---

## Why we use it

Mainly for Lazy Loading.

Example:

```csharp id="h9s2k4"
bus.Trips
```

Trips load automatically when accessed.

Without `virtual`, lazy loading cannot happen.

---

IMPORTANT : If you don't use `virtual`, then **lazy loading** will not work. âś…

But you can still fetch Trips data using:

```csharp id="q9w2m7"
.Include(b => b.Trips)
```

So:

* With `virtual`
  â†’ EF can automatically fetch related data (lazy loading)

* Without `virtual`
  â†’ You must fetch manually using `Include()` or explicit loading.

Yes, if Lazy Loading is properly enabled and `virtual` is used, then usually you do not need `Include()`.

Example:

```csharp id="j2x8v4"
var bus = context.Buses.First();

var trips = bus.Trips;
```

When `bus.Trips` is accessed, EF automatically loads trips from DB.

But this works only if:

* `virtual` is used
* Lazy loading proxies are enabled

Otherwise you must use:

```csharp id="k7m1q5"
.Include(b => b.Trips)
```

exact reply : â€śWe use `virtual` mainly to support Lazy Loading in Entity Framework Core.
It allows EF Core to create a proxy child class at runtime, so when we access related data like `bus.Trips`, EF can automatically fetch that data from the database if it is not already loaded.â€ť

Purpose of the child/proxy class:
It monitors access to virtual navigation properties.
Example:
bus.Trips

When you access this property, the proxy class detects it and automatically loads Trips data from database.

So the proxy class is created to add automatic behavior like Lazy Loading without changing your original class.

# Exact Definition of Binding Attribute

A binding attribute tells ASP.NET Core from where to get/bind parameter values in an HTTP request.

---

# 1. `[FromQuery]`

Gets data from query string.

URL:

```txt id="m4x8q1"
api/bus?min=10&max=50
```

Code:

```csharp id="p7n2v5"
[HttpGet]
public IActionResult Get([FromQuery] int min, [FromQuery] int max)
{
    return Ok();
}
```

---

# 2. `[FromBody]`

Gets data from request body.

JSON:

```json id="u1m9x4"
{
  "name":"Volvo"
}
```

Code:

```csharp id="r5m2x8"
[HttpPost]
public IActionResult Create([FromBody] BusRequestDto dto)
{
    return Ok();
}
```

---

# 3. `[FromRoute]`

Gets value from URL route.

URL:

```txt id="n7q1v4"
api/bus/5
```

Code:

```csharp id="p4m8x2"
[HttpGet("{id}")]
public IActionResult Get([FromRoute] int id)
{
    return Ok();
}
```

---

# 4. `[FromForm]`

Gets data from form-data.

Code:

```csharp id="w5q2n8"
[HttpPost]
public IActionResult Upload([FromForm] string name)
{
    return Ok();
}
```

Mostly used in file upload forms.

---

# 5. `[FromHeader]`

Gets value from request headers.

Code:

```csharp id="m8x2q4"
[HttpGet]
public IActionResult Get([FromHeader] string token)
{
    return Ok();
}
```

---

# 6. `[FromServices]`

Gets service directly from Dependency Injection container.

Code:

```csharp id="k5v1n7"
public IActionResult Test([FromServices] IBusService service)
{
    return Ok();
}
```

# Response Methods + Status Codes

| Method           | Status Code |
| ---------------- | ----------- |
| `Ok()`           | 200         |
| `Created()`      | 201         |
| `NoContent()`    | 204         |
| `BadRequest()`   | 400         |
| `Unauthorized()` | 401         |
| `NotFound()`     | 404         |




| Method            | Status Code | Example                                 |
| ----------------- | ----------- | --------------------------------------- |
| `Ok()`            | 200         | Data fetched successfully               |
| `Created()`       | 201         | New Bus created successfully            |
| `NoContent()`     | 204         | Delete successful but nothing to return |
| `BadRequest()`    | 400         | Invalid input/model validation failed   |
| `Unauthorized()`  | 401         | User not logged in or invalid token     |
| `NotFound()`      | 404         | Bus with given ID does not exist        |
| `StatusCode(500)` | 500         | Server/database exception occurred      |

Example of `BadRequest()`:

```csharp id="m4x8q1"
if(dto == null)
    return BadRequest();
```

Example of `NotFound()`:

```csharp id="p7n2v5"
if(bus == null)
    return NotFound();
```

Example of `500`:

```csharp id="u1m9x4"
catch(Exception)
{
    return StatusCode(500);
}
```
